Episode 029

Simplifying the BFF with ProxyKit

ASP.NET Core: From 0 to overkill

What we have so far

  • A stable (yet overkill) base infrastructure
  • A very basic application in terms of features

Next steps

  • Get back to implementing application features

But before

  • The way we implemented the BFF in episode 20 will be a pain to maintain
    • We're basically manually proxying requests to the backing APIs (Group Management only so far)
    • Any time we need to change something in the APIs, we need to adjust the code in the BFF
    • This is useful when we want to create customized requests/responses for the specific frontend, but that's not always the case
    • We did it like that originally to play with the HttpClient, but it's time to simplify

In this episode

  • Introduce ProxyKit to the BFF, to simplify integration with backing APIs
    • Any request targeting a backing API will be automatically proxied to it
    • This move requires some adjustments, as we move from being totally MVC bound
      • Authentication enforcement must move from being based on MVC filters to more general ASP.NET Core middlewares
      • Same for anti-forgery token validation
  • Very quick look at ProxyKit vs Ocelot
